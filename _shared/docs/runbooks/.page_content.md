# Runbooks

Operational procedures for the Order Service. These runbooks are intended for on-call engineers and should be followed step-by-step during incidents or planned operations.

> [!warning]
> If you're responding to an active incident, go straight to [Incident Response](page:"Incident Response"). Don't start with deployment or rollback unless the incident cause is confirmed.

| Runbook | When to use |
| --- | --- |
| [Deployment](page:"Deployment") | Planned releases to staging or production |
| [Rollback](page:"Rollback Procedure") | Reverting a bad deploy |
| [Incident Response](page:"Incident Response") | Responding to production alerts |
